# KEMI FLY

케미가 하늘섬 사이를 날며 수정과 AI 구름을 모으는 모바일/웹용 원터치 비행 게임입니다.

## 실행

```bash
npm run dev
```

브라우저에서 `http://127.0.0.1:5173`을 엽니다.

## 조작

- 모바일/웹: 탭 또는 클릭으로 상승
- 키보드: `Space` 상승, `Esc`/`P` 일시정지, `M` 사운드 토글, `F` 전체화면, `Enter` 시작/재시작

## 기능

- 충돌 즉시 게임오버가 아닌 에너지 감소 방식
- 모바일 세로/가로 및 데스크톱 화면에 맞춰 캔버스와 HUD 자동 전환
- 초록/금색 `+` 링으로 표시되는 수집 아이템, 붉은 `!` 표시가 붙은 장애물, 용암 포탈 트랩
- 수정 점수, AI 구름 회복, 보호막, 자석, 감속, 활공, 부스트, 근접 회피 보너스, 콤보/피버 보상
- 아이템 수집 개수에 따라 Lv1~Lv6까지 케미 외형, 피격 내성, 실드 한도가 성장
- 수정 수집, 근접 회피, 콤보, 레벨, 용암 도전 등으로 순환되는 미션과 메뉴 목표 제시
- 거리별 항로 페이즈가 HUD와 배너로 표시되어 현재 진행 구간과 다음 플레이 의도가 드러납니다.
- 거리 진행에 따라 `STAGE 1 초원`, `STAGE 2 사막`, `STAGE 3 하늘섬` 배경으로 전환되고, 각 스테이지에서 용암 포탈을 먹으면 고속 용암 챌린지가 별도로 발동됩니다.
- 수집 아이템이 가끔 짧은 트레일로 등장해 플레이어가 자연스럽게 비행 경로를 선택하도록 유도합니다.
- 에너지가 낮으면 하트/구름/실드 계열 회복 아이템이 우선 등장해 억울한 실패를 줄입니다.
- 레벨업 순간 배너, 용암 챌린지 배너, 게임오버 하이라이트로 성장/실패 피드백 강화
- 5콤보 단위로 에너지/점수/피버 보상이 주어지고 10콤보 단위로 실드 보상이 붙습니다.
- 화면 오른쪽 위험 예고 마커로 다가오는 장애물의 높이를 미리 읽을 수 있습니다.
- 게임오버 요약에 다음 판에서 바로 적용할 수 있는 맞춤 코칭 팁을 제공합니다.
- 게임오버 화면에서 기록 저장, 다시하기, 명예의 전당, 데미안 지원하기 동선 제공
- 첫 접속 메뉴에서 게임하기, 명예의 전당, 데미안 지원하기를 바로 선택할 수 있습니다.
- MP3 배경음악과 로컬 WAV 효과음
- 메뉴 배경 이미지를 살짝 확대해 천천히 패닝하고 은은한 반짝임 연출
- 모바일 세로 화면에서도 하늘섬이 보이도록 원거리 배경 크롭을 보정하고 루프 이동/바람선으로 비행감 표현
- 진행 거리에 따라 장애물 속도와 복합 패턴이 강화되고, 용암 포탈 트랩을 먹으면 일정 시간 고속 용암 지대와 점수 배율이 함께 적용
- 후반부에는 화염 암석이 좌하단으로 낙하하고 세로형 수정 장애물이 위아래로 움직여 회피 난이도가 올라갑니다.
- 용암 구간을 끝까지 버티면 돌파 보너스 획득
- 먹는 아이템은 밝은 보상 아이콘과 초록/금색 `+` 링, 장애물은 돌/보라/용암 계열과 붉은 `!` 표시로 구분
- 브라우저 localStorage 기반 명예의 전당 Top 10
- Vercel 배포 시 Vercel Blob 기반 명예의 전당 Top 10 영속 저장
- `manifest.webmanifest`와 서비스 워커 기반 설치형 웹앱 준비
- Playwright 검증용 `window.render_game_to_text`, `window.advanceTime(ms)` 제공

## 보안 및 데이터

- 배포 환경에서는 같은 출처의 `/api/leaderboard`만 호출합니다.
- 외부 오디오/이미지 CDN 없이 로컬 에셋, 로컬 MP3, 브라우저 Web Audio만 사용합니다.
- 입력 이름은 클라이언트에서 길이와 문자 범위를 제한하고, 화면 출력 시 DOM 삽입 대신 Canvas 텍스트로만 렌더링합니다.
- 데미안 지원 링크는 사용자가 직접 누른 경우에만 새 창으로 열며, 열린 창이 게임 페이지를 제어하지 못하도록 차단합니다.
- 명예의 전당 기록은 서버 API에서 다시 검증한 뒤 Vercel Blob에 저장합니다.
- Blob 저장소가 준비되지 않은 로컬 개발 환경에서는 기존 localStorage로 자동 폴백합니다.
- 개발 서버는 `127.0.0.1`에만 바인딩합니다.
- 배포 시 `BLOB_READ_WRITE_TOKEN`은 Vercel 환경변수로만 설정하고 저장소에 커밋하지 않습니다.

## 배포 준비

1. Vercel 프로젝트에 Blob Store를 연결합니다.
2. Vercel 환경변수 `BLOB_READ_WRITE_TOKEN`을 설정합니다.
3. GitHub 저장소와 Vercel 프로젝트를 연결한 뒤 Preview 배포부터 검증합니다.
